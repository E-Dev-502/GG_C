Changing "undefined" text

You can put "||" symbols after your function.

(fake-tag-s)alert("Hello!");(fake-tag-e) gives undefined.

(fake-tag-s)alert("Hello!") || "Hello!";(fake-tag-e) gives Hello!.

Echo ↓
Echo

(fake-tag-s) ?{ (Echo ID) "String" }? (fake-tag-e)

You can use echo command in loops.


Example

(fake-tag-s)
for (var i = 0; i < 8; i++) {
?{(58) i ?inTimeout:false}?
}
(fake-tag-e)
<p>?(58)?</p>

Output

{- for (var i = 0; i (lt) 8; i++) { ?{(58) i ?inTimeout:false}? } -}

?(58)?



Descriptions

Colored Text Description
?{(58) i ?inTimeout:false}? Echo command start and end tag.
?{(58) i ?inTimeout:false}? Echo id parentheses.Please put space after parentheses.
?{(58) i ?inTimeout:false}? Echo ID.You can give any number.
?{(58) i ?inTimeout:false}? Prinnt text.Any.
?{(58) i ?inTimeout:false}? Use it for timeout functions.Set true/false or leave it empty.Default value is false.
?(58)? Prinnt area tags.Do not put in {- -} tags.
?(58)? Echo ID for print.

(lt) and (gt)

Sometimes the < or > symbols do not work correctly. Instead, use the symbol (lt) or (gt) in the code.

Example : (fake-tag-s) for (var i = 0; i < 8; i++) { console.log(i); } (fake-tag-e) code maybe can gives error.

But : (fake-tag-s) for (var i = 0; i (lt) 8; i++) { console.log(i); } (fake-tag-e) code working correctly.
